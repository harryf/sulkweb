Sulk Game Data

Sulk /data folder — what’s inside and what it’s for

(Everything here is engine-agnostic; paths are shown relative to data/)

Top-level sub-dirPurposeTypical consumption in a fresh engine
fonts/Bundled TTF typefaces plus meta-files.Load each font once at startup, map to in-game styles (dialog, HUD, dice).
images/Engine-chrome graphics (buttons, splash, app icon). These never change with theme selection.Import as UI sprites; fits in any display resolution because each button is self-contained PNG with alpha.
missions/All mission plug-ins (MISH_*.py) plus a template and nested family folders (beta/, space_hulk/, debug/).Dynamic discovery at launch; each file is a small rules script implementing the template described in MISSIONS_INFO.
sounds/WAV clips for nearly every UI or combat event; SOUNDS_INFO states copyright.Load lazily on first use or preload into a sound-bank; each maps one-to-one to a gameplay event.
themes/Skinnable artwork packs. The root holds THEMES_INFO; each child dir (only default/ is shipped) is a self-contained atlas of piece and board-tile PNGs.Allow runtime theme override by prepending an alternate theme folder to the search path; if an asset is missing, fall back to themes/default/.

1. fonts/

bash
Copy
fonts/
├── FONTS_INFO          # explains licences
├── fudd.ttf
├── fudd.txt            # licence text for Fudd font
├── kimberley.ttf
└── kimberley.txt       # licence text for Kimberley font
  • Fudd (Matthew Welch) and Kimberley (Ray Larabie) are free fonts; each has an explicit licence TXT.
  • The engine uses them at different sizes for: large headlines, dialogs, HUD clocks and dice faces.

2. images/ (UI chrome)

All 48 × 48 ish PNGs with premultiplied alpha (button faces) or colorkey transparency (legacy cursor sprites).

  • Naming convention:

    thing_<state>.png where state ∈ {enabled, pressed, disabled}

  • Files shipped cover every global and per-piece button (shoot, door, self-destruct, etc.), so a port can bind button logic to one sprite sheet without slicing.
  • Also includes:
    • splash.png – title screen displayed during asset loading.
    • icon.png – window/task-bar icon.

3. missions/


missions/
├── MISSIONS_INFO           # long template (included in prompt)
├── beta/                   # family folder
│   ├── MISH_beta_1.py
│   └── MISH_beta_2.py
├── debug/
│   └── MISH_debug_1.py
└── space_hulk/
    ├── MISH_space_hulk_1.py .. _6.py
  • Discovery rule: walk sub-directories, import every MISH_*.py.
  • Template (MISSIONS_INFO) spells out the contract:
    • Constants: NAME, FAMILY, NUM, AUTHOR, FLAGS, BLIPS, BLIP_SET…
    • BOARD layout is a nested tuple of sections → squares, where each square may embed a dict of feature codes (DOOR, EXIT, etc.).
    • Optional callbacks (pre_deploy_rule, post_action_script, victory_check, …) give scenario-specific tweaks.
  • If two modules share the same FAMILY + NUM, load fails — enforced during import.

5. themes/default/

A full sprite atlas; any other theme must replicate the same filenames to be valid.

Critical dimensions

Base spriteMeaningConsequences
square_corridor.png1 logical board squareAll “square_*.png” must match this exactly.
mini_square.png1 miniature map tileMust be at least 2 px wide; ratio mini : full must stay constant across themes (20–30 to 1 suggested).

File groups

  • Board tiles – square_corridor.png, square_room.png
  • Mini-map – mini_square.png, mini_blip.png, mini_stealer.png, mini_terminator.png, mini_entryexit.png
  • Pieces & blips – terminator_*, stealer.png, blip.png, blip_1.png…

    (Many terminator poses allow damage animation.)

  • FX & overlays – flames, muzzle flashes, facing arrow, selection rings, sectionline.
  • Markers – jam, overwatch, convert, damage.
  • Features – doors open/closed, bulkhead, crate, ducting, rubble, cat, entry/exit.

Theme rules (from THEMES_INFO)

  1. Omit a file → engine falls back to themes/default/.
  1. Transparency must match the original (RGB key or alpha channel as appropriate).
  1. Mini : full scale must be consistent theme-wide.
  1. Keep colour depth, but resolution agnostic – engine draws pixel-perfect.

6. Licence meta-files

  • FONTS_INFO – credits and redistribution limits for Fudd & Kimberley.
  • SOUNDS_INFO – calls out three GPL-v2 UI sounds; everything else is Public Domain.
  • THEMES_INFO – technical constraints for creating additional themes.

Porting tips

  • Treat themes as a pluggable “asset bundle”. Your asset loader should search:
php-template
Copy
<chosen theme>/<filename>
→ default/<filename>
→ engine-error if missing
  • Images/ never themed, so can stay in one atlas.
  • Mission import can be retained verbatim (dynamic Python) or converted to JSON/YAML mirroring the template keys.
  • For sounds: map the filenames to enum events, store licence flag alongside for later packaging.

With this map you can replicate Sulk’s resource pipeline (fonts → draw, sounds → event bus, images → sprite sheets) in any engine without digging through the original loader code.
